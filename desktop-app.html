⏺ Record your own audio/video

🗄 Storage Backend

Where your uploaded bytes get parked. Encryption + sharing are identical either way — only storage differs. Your Hive key stays your identity; a Pinata key is just a storage credential (never sent to this server).

GATE URL
PINATA JWT (scoped to pinFileToIPFS)
GATEWAY (for share links)

A pinFileToIPFS-only key can only burn pin quota if leaked — it can't read, delete, or touch billing, and you can revoke it in Pinata anytime. You pay Pinata directly; no Hive fee, no v4call fee. (Encrypted-at-rest storage of this key is a planned future option.)

📌 Pinata management key

This is a separate, broader-scoped key from the pin-only key you paste to post files in a room. It can list, delete (unpin), and read storage usage — so it carries more risk if it leaks. It is stored only in this browser and is never sent to this server.

In Pinata → API Keys → New Key, tick exactly these scopes (leave the rest off to keep the blast radius small):
  • pinFileToIPFS — upload from this tab
  • pinList — list your pins
  • unpin — delete pins
  • userPinnedDataTotal — show storage used

⬆ Upload to your Pinata

Pins to your own Pinata account — you pay Pinata directly, no Hive or v4call fee. Choose plaintext (shareable link) or encrypt to a Hive key (you also get a downloadable encrypted bundle for later use).
